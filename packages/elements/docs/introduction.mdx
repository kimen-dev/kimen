# Kimen component workshop

**Kimen Elements** is the implemented Web Components foundation for Kimen's
planned, multi-protocol GenUI system. Today it provides standards-first
`<ki-*>` elements, generated machine contracts and token-driven theming in
any host. The neutral catalog, guardrail and protocol adapters come later.

This Storybook is the **component workshop**: every published component with
its props as live controls, its docs generated from the component contract
(the TypeScript types + JSDoc are the single source of truth), and an
accessibility panel wired to axe-core.

## How to read a component page

- **Playground** — every prop exposed as a control; slots shown as text.
- **Named stories** — one story per API axis (variants, tones, sizes…), so
  each design decision is visible in isolation.
- **Docs tab** — autogenerated from the manifest: props, events, slots,
  parts and CSS custom properties. If something is undocumented there, that
  is a build failure, not a docs gap.

## Theming in the toolbar

The **Scheme** toolbar switches light / dark / auto — it drives the same
`data-ki-color-scheme` attribute a consuming page would use. See
*Docs → Tokens & theming* for the full model.

## Where things live

  violations, packaging checks and bundle budgets). Mutation gating is being
  added during project-integrity hardening.

## For agents

Kimen treats machine legibility as a deliverable: components already produce
when-to-use / when-NOT-to-use metadata, a generated manifest and `llms.txt`.
A Zod-validated schema of what agents may emit is planned for the catalog
phase. An API a capable agent cannot wire correctly from its description
alone is a defect — file it as one.
