## Summary

Complete the Art. I machine-surfaces pipeline beyond Stencil's docs-json:
derive the community-standard Custom Elements Manifest
(`custom-elements.json`) and an `llms.txt` language-model summary from the
component contract, carry when-to-use / when-NOT-to-use guidance as dedicated
JSDoc tags (`@whenToUse` / `@whenNotToUse`), enforce documentation
completeness at build time, and verify all committed surfaces with a new
deterministic `surfaces-sync` gate (tokens-sync precedent). Technical
approach: zero new dependencies — a small ESM library
(`packages/elements/scripts/agent-surfaces.mjs`) maps the already-generated
`generated/docs.json` (normalized per FR-008: no timestamp, no absolute
paths) into both surfaces as a pure JSON→text transformation, chained into
the existing `@kimen/elements` build target so every future component (003 to
016) inherits the surfaces with zero wiring (FR-007). Details and rationales
in [research.md](./research.md); emitted formats in
[contracts/agent-surfaces-contract.md](./contracts/agent-surfaces-contract.md).

## Technical Context

**Language/Version**: Node ≥ 22 ESM (`.mjs`) for the generator scripts (the
`packages/tokens/build.mjs` precedent); TypeScript strict for the test files.

**Primary Dependencies**: none — zero new runtime AND zero new build-time
dependencies. The mapping is docs.json → JSON/text with `node:fs`,
`node:path` only (pnpm supply-chain policy: `minimumReleaseAge` 7d,
`trustPolicy: no-downgrade`; the cheapest dependency is the one not added).

**Storage**: N/A (committed generated files are the artifact).

**Testing**: Vitest 4, node environment, via a new
`packages/elements/vitest.node.config.ts` project chained into the existing
`test` target. Test files are `*.spec.ts` under `packages/elements/scripts/`
(the traceability gate only counts `*.spec.ts|*.spec.tsx|*.e2e.ts` under
`packages/`), marker `// @spec:017-agent-surfaces`, S-IDs in test titles.
No browser suite: no rendered surface exists (research.md D7).

**Target Platform**: build environment only (Node ≥ 22, per repo `engines`).
Generated artifacts target machine consumers (editors, agents, LLMs).

**Project Type**: infrastructure — build pipeline of `packages/elements`
plus one gate in `scripts/gates/gates-suite.sh`.

**Performance Goals**: surface generation adds < 1 s to the elements build
(single-pass JSON transform); no runtime cost, no size-limit entries (the
artifacts ship as package metadata, not as loadable code).

**Constraints**: byte-determinism (FR-006/S6: no timestamps, no absolute
paths, stable serialization, LF endings); all new files pass the existing
deterministic layer (biome format, ESLint, tsc, knip) without weakening any
rule; generated outputs live where biome/ESLint/knip already exclude
generated content (`packages/elements/generated/`) or are inert to them
(`llms.txt`); single writer on `feat/agent-surfaces`.

**Scale/Scope**: 6 approved-pending scenarios (S1–S6); one library module +
one CLI orchestrator + one preamble template; four committed artifacts
(normalized `docs.json`, `generated/custom-elements.json`,
`packages/elements/llms.txt`, root `llms.txt`); 14 downstream component
specs (003–016) inherit the T019 obligation this feature makes satisfiable
(FR-009).

**Structure Decision**: generator scripts live inside `packages/elements`
(the package whose build they extend — `packages/tokens/scripts/` precedent),
not in root `scripts/` (which holds repo-wide gates) nor `tools/` (Nx

so the existing biome/ESLint/knip exclusions apply with zero config churn;
`llms.txt` is text that no formatter or linter touches.
